5.2.1 2.1 Modelo digital de superficie

Trabajamos con datos de DSM (Modelo Digital de Superficie) que provienen de Copernicus (Agencia Espacial Europea) almacenados en Open Land Map.

Usamos un Cloud Optimized GeoTIFF (GeoTIFF Optimizado para la Nube), un formato dise√±ado para facilitar el trabajo en la nube, junto con el m√≥dulo rioxarray para importar solamente los datos necesarios de manera eficaz. En lugar de importar todos los datos globales (varios terabytes), importamos √∫nicamente nuestra √°rea de inter√©s mediante lazy loading (carga perezosa). Esto significa que no descargamos los datos hasta haberlos filtrado, usando as√≠ solamente la memoria y poder computacional esencial.

Mostrar c√≥digo 
# 1. Conectar al cat√°logo de Microsoft Planetary Computer
# Planetary Computer es una plataforma que provee acceso gratuito a datasets globales
catalog = pystac_client.Client.open(
    "https://planetarycomputer.microsoft.com/api/stac/v1",
    modifier=planetary_computer.sign_inplace,  # Firma autom√°ticamente las solicitudes
)

# 2. Reproyectar las cuencas al CRS del raster (EPSG:4326)
# Necesitamos esto para buscar los datos correctos en el cat√°logo
cuencas_4326 = cuencas.to_crs("EPSG:4326")

# 3. Obtener el bounding box (caja delimitadora) de nuestras cuencas
# bbox define el √°rea geogr√°fica de inter√©s: [min_x, min_y, max_x, max_y]
bbox = cuencas_4326.total_bounds

# 4. Buscar el Modelo Digital de Superficie (DSM) de Copernicus para nuestra √°rea
# cop-dem-glo-30 es el mismo dataset global de elevaci√≥n que us√°bamos de OpenLandMap
search = catalog.search(
    collections=["cop-dem-glo-30"],  # Copernicus DEM Global 30m
    bbox=bbox,  # Limitamos la b√∫squeda a nuestro √°rea de inter√©s
)

# Obtener los items (archivos) que cubren nuestra √°rea
items = list(search.items())

# 5. Cargar el DSM usando chunks para procesamiento eficiente en memoria
# odc.stac.load descarga y ensambla autom√°ticamente los tiles necesarios
dsm = load(
    items,
    bbox=bbox,
    chunks={"x": 1024, "y": 1024},  # Procesamiento por bloques para eficiencia
)

# 6. Extraer la banda de elevaci√≥n y convertir a DataArray para usar con rioxarray
# El resultado de load() es un Dataset; necesitamos el DataArray de elevaci√≥n
dsm = dsm["data"].squeeze()  # squeeze() elimina dimensiones extra

# 7. Recortar el raster a las geometr√≠as exactas de las cuencas
dsm_clipped = dsm.rio.clip(cuencas_4326.geometry)

# 8. Reproyectar el √°rea recortada al CRS de trabajo (POSGAR 7)
# Esto es m√°s eficiente porque reproyectamos solo el √°rea de inter√©s
dsm_clipped_reproj = dsm_clipped.rio.reproject(dst_crs=USE_CRS)

Los datos de elevaci√≥n son un ejemplo de datos continuos, lo que significa que los valores pueden tomar cualquier n√∫mero dentro de un rango (incluyendo decimales). Estos valores se almacenan como floats (n√∫meros de punto flotante) en vez de enteros, permitiendo representar mediciones precisas como 15.7 metros o 23.42 metros de elevaci√≥n.
